# Producto Visible

> Objetivo: dejar de medir el avance solo por logica interna y empezar a medirlo por lo que el administrador y el tenant pueden ver, entender y operar directamente en la interfaz.

## Que debe poder verse

1. Donde se da de alta una empresa.
2. Donde se le asigna branding.
3. Donde se le asignan modulos o servicios.
4. Donde se define su admin inicial.
5. Que vera esa empresa al iniciar sesion.
6. Como se diferencia visualmente del backoffice global.
7. Como aparecen `Control de Acceso` y `Redes` cuando estan habilitados.

## Regla de trabajo

Una mejora no se considera cerrada solo por tocar codigo o corregir documentos. Tiene que producir una diferencia visible de producto.

## Alcance de esta mini-etapa

- `Configuracion` como tablero de producto visible.
- `Empresas` como consola de tenant operable.
- `Servicios y paquetes` como lectura clara del producto vendido y del runtime real.
- `Tema` como branding visible y reutilizable.
- `Usuarios` y `Roles` como espacio tenant real.
- `Sidebar` como evidencia del menu por empresa.
